## Google Drive integration â€” discussion only

Three levels of integration, each with different complexity:

### Level 1 â€” URL import (simplest, zero auth)
Google Sheets "File â†’ Share â†’ Publish to web â†’ CSV" generates a public URL like `https://docs.google.com/spreadsheets/d/SHEET_ID/pub?output=csv`. Already handled by the URL fetch feature added today. User copies that link, pastes it into the URL box, clicks Fetch. No OAuth, no API keys. Works now.

**Tradeoff**: requires the user to publish the sheet publicly. Read-only import only.

### Level 2 â€” Google Sheets API with OAuth (medium)
Register a Google Cloud OAuth2 client (public client_id for a web app, no secret needed on the front end). Add a "Connect Google Drive" button that opens the OAuth popup. After auth, use the Sheets API to list/read spreadsheets. This allows reading private sheets.

**Tradeoff**: requires the user to set up a Google Cloud project OR you host a shared client_id. The Cloudflare Worker could proxy the token exchange. Moderate complexity (~200 lines + worker changes). The biggest complexity is managing the token lifecycle (refresh tokens in localStorage, expiry handling).

### Level 3 â€” Google Drive as a persistence backend (complex)
Replace or augment the current GitHub-based `.enc` file storage with Google Drive. The app already uses GitHub API for all reads/writes to `.enc` files. Drive has a similar REST API. This would let users who don't have a GitHub account use the app.

**Tradeoff**: very high complexity â€” Drive file IDs are not human-readable paths like GitHub paths, conflict resolution is different, the Drive API response format is entirely different. Would essentially be a second storage adapter. Not recommended unless GitHub persistence becomes a pain point.

### Recommendation
Level 1 is already live with today's URL fetch feature. Level 2 (OAuth import from private sheets) is reasonable to add if you regularly maintain a Google Sheet of greenhouse plants â€” the main work is the OAuth popup + a `fetchSheetRows()` function. Level 3 is not worth the complexity given the working GitHub backend.
