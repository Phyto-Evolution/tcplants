### 3. Add all JS — insert before `// ── End IoT Panel` line

Key functions to implement:

**`_fwOpen()`** — adds `.on` to `#fw-studio`, calls `_fwPinsInit()`, calls `_fwUpdate()`

**`_fwClose()`** — removes `.on` from `#fw-studio`

**`_fwGet(id)`** — returns `document.getElementById(id)?.value || ''`

**`_fwChecked(id)`** — returns `document.getElementById(id)?.checked || false`

**`_fwPins`** — module-level array: `[{label, type, gpio, unit}]`, start with 2 examples

**`_fwPinsInit()`** — if `_fwPins.length === 0`, seed with `[{label:'Relay 1', type:'relay', gpio:'2', unit:''}, {label:'Temp', type:'dht', gpio:'4', unit:'°C'}]`, then call `_fwRenderPins()`

**`_fwRenderPins()`** — sets `#fw-pin-list` innerHTML with one `.fw-pin-row` per pin, each with 4 inputs (label/type/gpio/unit) + remove button

**`_fwPinUpdate(i, field, val)`** — updates `_fwPins[i][field]`, calls `_fwUpdate()`

**`_fwAddPin()`** — pushes `{label:'', type:'relay', gpio:'', unit:''}`, re-renders, updates

**`_fwRemovePin(i)`** — splices, re-renders, updates

**`_fwUpdate()`** — debounced 200ms: calls `_fwGenerateCode()`, sets `#fw-code-display` innerHTML to `_fwAddLineNums(_fwHighlight(code))`, updates filename + linecount

**`_fwHighlight(raw)`** — **tokenizing state machine** (NOT regex replace — do it properly):
- Walk character by character
- Handle: `//` single-line comments → `.fw-cm`
- Handle: `/* */` block comments → `.fw-cm`  
- Handle: `"..."` string literals → `.fw-cs`
- Handle: `'...'` char literals → `.fw-cs`
- Handle: `#` at line start → preprocessor directive to end of line → `.fw-cp`
- Handle: word tokens — check against keywords set (`void int float double bool char long unsigned const struct return if else for while do switch case break continue new delete nullptr true false String byte word size_t uint8_t uint16_t uint32_t`) → `.fw-ck`; Arduino constants (`HIGH LOW INPUT OUTPUT INPUT_PULLUP LED_BUILTIN`) → `.fw-ca`; if followed by `(` → `.fw-cf`; else plain
- Handle: numeric tokens → `.fw-cn`
- All other chars: HTML-escape and emit raw

**`_fwEsc(s)`** — `s.replace(/&/g,'&amp;').replace(/</g,'&lt;').replace(/>/g,'&gt;')`

**`_fwAddLineNums(html)`** — split by `\n`, wrap each in `<span class="fw-ln">...</span>`, rejoin with `\n`

**`_fwCopy()`** — `navigator.clipboard.writeText(_fwGenerateCode())` + toast

**`_fwDownload()`** — Blob of `_fwGenerateCode()`, trigger `<a>` download with filename `${name}.ino`

**`_fwGenerateCode()`** — reads ALL form fields, builds and returns the complete `.ino` string as a template literal. The generated code must be:
- A complete, compilable Arduino `.ino`
- Header comment: device name, date, board, worker, features, library list (conditional on which sensors/features are enabled)
- Platform detection `#if defined(ESP32)` block
- Conditional `#include` for OLED (Adafruit_SSD1306), OTA (ArduinoOTA), DHT (if any dht pin), DS18B20 (if any ds18b20 pin)
- `#define` block with all config values baked in
- Config struct + EEPROM/Preferences load/save (same as original firmware)
- `DEFAULT_PINS[]` array baked from the pin builder (used as fallback if server fetch fails)
- `setup()`: serial begin → boot message → optional OLED splash → loadConfig → connectWiFi → optional OTA setup → register if no deviceId → fetchPinConfig → setupPins → optional OLED IP display
- `loop()`: handleSerialConfig → optional ArduinoOTA.handle → heartbeat → command poll
- All functions from the original firmware: handleSerialConfig, connectWiFi, ensureWiFi, httpPost, httpGet, registerDevice, fetchPinConfig, loadDefaultPins, setupPins, readPin (with DHT/DS18B20 branches if those pin types exist), sendHeartbeat, pollCommands, executeCommands, confirmCommand
